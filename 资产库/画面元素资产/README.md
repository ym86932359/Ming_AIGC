## 编号规则

`element_id`使用`ZERO-ECHO-ELEM-类别-名称-NN`，例如：

```text
ZERO-ECHO-ELEM-SIGN-ZERO-DOOR-01
ZERO-ECHO-ELEM-WEATHER-RAIN-MEDIUM-01
ZERO-ECHO-ELEM-SETDRESS-BAR-BOTTLE-GROUP-01
```

## 调用规则

1. 同一元素只保存一份正式资产卡；场景卡记录默认位置，镜头调用表记录本镜状态。
2. 固定元素不得在镜头间随机消失、换边或改变数量；可变元素必须写入合法状态和恢复条件。
3. 有人物直接拿取、启动或改变剧情结果的物件，应升级为道具，不在本目录复制第二份正史。
4. 可见文字必须记录精确内容、字体、语言、位置和允许的发光状态，禁止AIGC随机生成伪文字。
5. 每个进入正式镜头的元素必须关联`material_id`、`palette_id`、可见镜号和当前视觉版本。

## 锁定条件

- [ ] 正侧背、尺寸、材质和场景坐标可复核。
- [ ] 固定/可变属性、状态机和镜头可见性已登记。
- [ ] 交互、VFX、声音和连续性责任清楚。
- [ ] 当前视觉资产、版本、状态和校验信息齐全。
