# 零点回声酒馆 - 人物模型全要素规范

> 本页定义“一个人物模型需要保存哪些数据”。具体人物值保存在角色人物卡，当前画风保存在[[个人创作/ai铭仔/世界观与视觉/零点回声酒馆 - 新国风赛博琉璃人物视觉风格规范]]，本页不重复角色内容。

## 一、身份与版本

| 必填字段 | 用途 | 例 |
| --- | --- | --- |
| `asset_id` | 人物文字资产唯一编号 | `ZERO-ECHO-CHAR-LINYE` |
| `bundle_id` | 关联包唯一编号 | `ZERO-ECHO-BUNDLE-LINYE-EP01` |
| `continuity_id` | 故事阶段连续性编号 | `LINYE-S1-EP01` |
| `visual_style_id` | 人物画风编号 | `ZERO-ECHO-VIS-NEO-CHINESE-CYBER-GLAZE-02` |
| `style_profile` | 风格规范双链 | 项目风格文件 |
| `version` | Markdown版本 | `v04` |
| `visual_asset_version` | 当前图片版本 | `v04` |
| `status` | 草案/待审/锁定/废弃 | `待审` |
| `story_stage` | 年龄、服装和伤痕阶段 | 第一季第1集 |

## 二、正史与身份来源

- 用户明确设定、正式剧本与连续性、真人/锁定身份原型、当前有效总卡、辅助三视图、版式案例必须分级。
- 每张参考图记录唯一职责与不继承内容。
- 冲突记录必须写明旧口径、当前正史、采用结果和废止资产。
- 风格文件永远低于人物身份、年龄、种族、伤痕和核心道具正史。

## 三、人物内核

- 姓名、角色功能、职业、阵营、故事阶段、叙事年龄和视觉年龄。
- 第一印象、性格特色、核心矛盾、核心欲望、核心恐惧和人物弧。
- 视线、呼吸、站姿、手势、步态、说话速度和压力动作。
- 与其他角色的关系及该阶段的情绪基线。

## 四、身体与身份锚点

| 要素 | 必须记录 |
| --- | --- |
| 脸与年龄 | 额头、颧骨、眼距、鼻唇、下颌、视觉年龄和禁止漂移 |
| 发型 | 发色、发际线、分缝、长度、侧面层次、背面闭合和动态 |
| 眼眸 | 天然底色、瞳孔、眼白、能力接口和发光禁区 |
| 服装轮廓 | 领口、肩线、腰线、下摆长度和前后闭合 |
| 核心道具 | 名称、尺寸、材质、方向、惯用手和剧情标记 |
| 身体比例 | 身高感、头身比、肩腰比、四肢、重心和体态 |

## 五、三视图

- 正面：脸、发型、服装中轴、左右配饰、道具固定位置和鞋头。
- 侧面：额鼻唇下颌曲线、耳前后发层、肩腰厚度、开衩、道具受力和鞋跟。
- 背面：后脑结构、服装闭合、肩背组件、腰带、下摆、鞋后跟和挂点。
- 四分之三：用于分镜身份确认；不得生成另一张脸。

## 六、面部微观细节

- 脸型、骨相、正脸比例、侧脸曲线、默认表情和表情禁区。
- 眼睛：眼裂、眼皮、眼尾、虹膜纤维、瞳孔、角膜高光、睫毛和目光习惯。
- 眉毛：粗细、弧度、眉峰、眉尾、毛流和颜色。
- 鼻部：鼻梁、鼻尖、鼻翼、鼻孔边缘、毛孔和受光。
- 唇部：唇峰、厚度、嘴角、唇纹、湿润度与唇色。
- 皮肤：冷暖、毛孔、绒毛、痣、雀斑、疤痕、年龄纹、妆面和高光分区。

## 七、发型

- 发根、主体、暗部、高光与挑染色值。
- 发质粗细、软硬、干湿、光泽和发束分组。
- 正面刘海与碎发、侧面耳前耳后层次、背面蓬度和发尾闭合。
- 风、雨、奔跑、静止和记忆空间中的允许变化。

## 八、服装与赛博琉璃

| 区域 | 结构字段 | 材质字段 | 连续性字段 |
| --- | --- | --- | --- |
| 领口 | 高度、交叠、开口、闭合 | 织物、金属、琉璃 | 正侧背一致 |
| 肩部 | 肩线、护片、活动量 | 黑钛框、琉璃厚度 | 左右数量固定 |
| 胸背 | 中轴、覆片、接口 | 织纹、粗糙度、折射 | 后背闭合成立 |
| 手臂/手腕 | 袖型、护臂、手套 | 弯折、磨损、透明度 | 惯用手不跳变 |
| 腰部 | 腰封、扣件、挂点 | 皮革、黄铜、数据纹 | 道具位置固定 |
| 裤装/裙摆 | 轮廓、开衩、裤脚 | 垂坠、厚度、风响应 | 长度不跳变 |
| 边缘 | 滚边、收口、线性高光 | 旧金、电子织线、琉璃 | 发光面积受控 |

琉璃额外记录：厚度、颜色、粗糙度、透射、折射、内部深度、数据纹、划痕、湿态、受力框架、暖冷光响应和禁用区域。

## 九、配饰与鞋履

- 每件配饰记录名称、数量、左右位置、尺寸、色值、材质、闭合方式和不可消失镜头。
- 鞋履记录正面、外侧、内侧、背面、鞋底、鞋跟、防滑纹、磨损、雨水和受力。
- “无”必须明确写出，禁止生成模型自由添加首饰、武器或品牌。

## 十、道具

- 剧情功能、尺寸比例、轮廓、材质、透明度、磨损和状态变化。
- 惯用手、握持点、手指受力、朝向、挂载方式和离手状态。
- 与角色服装、色卡、能力和场景光的反射关系。
- 每个道具必须链接唯一资产卡，不在角色人物卡复制完整道具正史。

## 十一、表情与动作

- 表情卡固定24格；每格记录编号、中文名、眉眼、眼睑、视线、鼻翼、嘴角、下颌、呼吸、泪膜和剧情用途。
- 动作记录预备、触发、峰值、余波和离镜状态；表情编号与动作状态必须同时出现在分镜连续性表。
- 能力、妆造或故障状态只改变正史允许区域，不重新设计脸与服装。

## 十二、色卡、光影与调色

- 全局色卡编号、角色色卡编号、继承色和角色专属色各自独立。
- 主光、辅光、左后轮廓光、右后轮廓光、眼神光和能力/数据光分别记录角度、色温、强度和用途。
- 曝光补偿、肤色IRE、头发/琉璃/金属高光上限、饱和度、自然饱和、对比、高光、阴影、黑位和肤色保护。
- 场景光可以改变观感，不得改写角色固有色、天然眼眸和琉璃类型。

## 十三、连续性与变体

- 临时雨水、污渍、轻伤和情绪不新建连续性编号，但写入场景状态。
- 年龄阶段、主发型、主服装轮廓、核心道具、不可逆伤痕或长期妆造变化必须新建连续性编号。
- 每个变体记录进入触发、退出条件、可逆性和继承自哪个基础模型。

## 十四、版本、审核与历史

- 当前人物卡、总卡、表情卡、提示词和呈现参数分别递增版本。
- 历史图不删除、不覆盖；关联包只加载当前有效版本。
- 每次审核记录同脸、年龄、三视图、材质、手部、道具、标签、裁切、特殊状态和用户结论。
- 技术审核通过只能设为`待审`；用户确认后才能设为`锁定`。

## 十五、标准调用

```text
调用 [CONTINUITY_ID] 人物模型。
加载人物关联包、人物资产卡、当前人物总卡、当前面部表情卡、呈现参数、项目风格配置、材质ID、道具ID和正史来源。
使用 [PERFORMANCE_STATE_ID] / E[两位编号] [中文情绪名]。
禁止从历史版本目录或文件夹相似图片中随机取图。
```

## 锁定检查

- [ ] 15类要素均填写或明确写“无”。
- [ ] 总卡与表情卡同脸、同年龄、同发型、同服装版本和同风格编号。
- [ ] 风格、材质、颜色和灯光分别描述，没有混成空泛美术词。
- [ ] 当前图片、历史版本、提示词、状态和双链一致。
- [ ] 用户确认允许进入正式分镜生产。
