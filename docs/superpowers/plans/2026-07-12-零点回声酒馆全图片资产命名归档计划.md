# 零点回声酒馆全图片资产命名归档 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task.

**Goal:** 将《零点回声酒馆》全部现有图片、尚未回写的相关生成图和技术审片帧，按画面内容、资产职责、版本与状态完成语义化命名，存放到 Obsidian 对应目录，并建立可检索、可追溯的图片总索引。

**Architecture:** 正式人物图归人物资产角色目录；场景、道具和空镜归宣传片制作资产；导演审片帧、字体检查图和中间转换图保留在工程目录但使用内容化文件名；废弃或未锁定视觉稿只进入角色 `历史版本`，不得成为下游默认调用源。所有路径通过 Obsidian README、人物关联包和一份全图片总索引汇合。

**Tech Stack:** Obsidian Markdown、YAML Frontmatter、Wikilinks、PNG/JPEG/WebP、SHA-256、macOS 文件工具、Ripgrep。

## Global Constraints

- 不覆盖现有图片，不删除历史图片；版本变更一律新增文件或无冲突重命名。
- 只归档与当前作品明确相关的生成图；《月陨》图片回到《月陨》人物目录，其他无关项目图片不导入。
- 人物当前有效资产与历史样张严格分开；历史图片必须写明废弃或非当前正史原因。
- 对被代码、Markdown、HTML 或 JSON 引用的图片先建立路径映射，再同步更新全部引用。
- 图片二进制迁移完成后校验文件可读、尺寸、SHA-256、文件名和引用目标。
- 用户并发新增的资产属于用户内容；执行前后重扫并增量纳入，不覆盖、不回滚。
