# PV01石砖终帧母图生成 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 从零生成一张文字准确、排版工整、光影统一的16:9《零号回声酒馆》石砖终帧母图，并在进入多角度制作前完成严格审核与Obsidian归档。

**Architecture:** 采用“单一母图先行”架构：GPT Image只生成一张完整终帧，所有汉字只出现一次；母图通过六字计数、逐字人工、中文识别、字距、光向和材质闸门后，才成为未来多角度资产的唯一图像身份源。本计划不生成A01—A04角度图。

**Tech Stack:** Obsidian Markdown、YAML Frontmatter、Wikilinks、GPT Image 2平台内建图像生成、macOS Vision文字识别、`sips`、SHA-256。

## Global Constraints

- 输出为单一16:9终帧，不做四格，不重复完整标题。
- 从零生成，不传入v01—v04图片作为参考。
- 主标题严格为六字`零号回声酒馆`，不含逗号、句号或其他标点。
- 副标题严格为四字`敬请期待`。
- 六字等高、等距、共用基线；“声”使用标准简体字形且只出现一次。
- 主标题采用方正整体石刻浮雕，不使用大量零散砖块拼字。
- 左上3000K琥珀主光让文字向右下投射实体阴影；字背隐藏光从右侧和下侧泄出，与投影呼应。
- 背光以暖象牙为主，只混入10%—15%故障青；不形成霓虹、LED或全墙青光。
- 母图目标为3840×2160 PNG；平台返回尺寸若不足4K，只能作为审片样张，不得冒充4K母版。
- 历史v01—v04保留但不调用；新资产从v05开始，失败迭代递增版本号且不覆盖。
- 当前目录不是Git仓库，不执行commit、merge或PR步骤。

### Task 3: 执行文字、排版和光影三重验收

**Files:**
- Verify: `宣传片/PV01-世界观先导片/制作资产/09-石砖终帧母图与角度资产/PV01-POST-E01-石砖终帧母图-v05.png`

**Interfaces:**
- Consumes: Task 2母图候选。
- Produces: `DIRECTOR-PASS / USER-REVIEW-PENDING`或`DIRECTOR-REJECTED`结论。

- [x] **Step 1: 建立文字识别闸门**

用macOS Vision只识别母图中央文字区域，输出标题候选、标题字符数、重复`声声`检查和副标题结果。

- [x] **Step 2: 运行机器辅助检查**

Expected:

```text
TITLE_SLOT_COUNT=6
DUPLICATE_SHENG=false
SUBTITLE_OK=true
```

机器识别只辅助计数，不能替代逐字人工审片。

- [x] **Step 3: 放大人工逐字检查**

按顺序确认：`零1｜号2｜回3｜声4｜酒5｜馆6`；确认无逗号；特别检查“声”的上部与下部结构、“酒”的三点水和“馆”的饣旁。

- [x] **Step 4: 排版检查**

确认六字等高、共用基线，任意相邻字距不得超过平均字距的110%或低于90%；无删除字后的空槽。

- [x] **Step 5: 光影检查**

确认左上主光、右下实体投影、右下字背泄光方向一致；背光不吞笔画，故障青不超过局部字背光的15%。

- [x] **Step 6: 记录结论**

只有全部通过才写`DIRECTOR-PASS / USER-REVIEW-PENDING`。任何文字或排版错误都写`DIRECTOR-REJECTED`，并递增版本从零生成，不基于失败图编辑。
