# PV01｜RunningHub无限画布65节点横向泳道重排设计

## 一、设计结论

采用用户确认的**方案A：横向资料泳道**。

- 分类区域从上到下分层，画布整体向下延伸。
- 每个分类内部从左到右排列，后续新增资料继续向右追加。
- 不同分镜使用独立横排，上下留出安全距离，但各分镜内部保持并列阅读。
- 所有图片、角色卡、道具、音频、分镜板和现有视频只移动、重命名或补充文字说明，禁止重新上传。
- Seedance生成视频进入独立区域；每个视频必须有镜号、Take、首帧、参数、提示词摘要、声音与审核状态说明。

## 二、当前画布审计

2026-07-12只读审计结果：

| 项目 | 当前事实 |
| --- | --- |
| 现有节点 | 65个 |
| 现有连线 | 1条：KF01 v02工作副本 → S01 Seedance视频节点 |
| 世界空镜 | 5个 |
| 场景关键帧与道具 | 13个 |
| 角色卡与A/B/C表演板 | 11个 |
| 四幕分镜板 | 4个 |
| 对白、环境与SFX | 18个 |
| 导演预演与片尾海报 | 2个 |
| 导演信息板 | 10个 |
| Seedance生产节点 | 2个：KF01 v02工作副本＋S01视频 |

现有节点分散为多个上下孤立簇，分类边界不连续；世界、人物、道具、分镜、音频和生产节点使用不同坐标原点，导致全局阅读路径不明确。

## 三、网格与安全距离

### 统一坐标合同

- 画布逻辑原点：`x=0, y=0`。
- 常规横向节距：`700px`。
- 常规纵向行距：`900px`。
- 分类区最小垂直间距：`1200px`。
- 任何两节点静态边界不得重叠；最小可见留白：横向`≥160px`、纵向`≥300px`。
- 3:4角色卡按最大高度约`600px`计算；横版场景、信息板和音频节点不得侵入相邻泳道。
- 同一横排统一基线；禁止出现几十像素的随意错位。

### 命名规则

不改变本地文件名，只调整RunningHub节点显示名，并统一加区域前缀：

```text
[00总控] [01世界] [02道具] [03角色-A] [03角色-B]
[04A-S01-S04] [04B-S05-S08] [04C-S09-S11] [04D-S12-S14]
[05对白] [05环境/SFX] [06生成视频] [07预演] [99历史]
```

## 四、区域坐标与节点归属

### 00｜导演总控与导航｜`y=0`

从左向右：

1. `00-导演总控与状态图例.png`
2. `01-资产地图与唯一职责.png`
3. `03-四幕14镜时间轴.png`

职责：先读项目规则、资产地图，再进入分镜时间轴。其余信息板下沉到对应专业区，不集中堆在总控区。

### 01｜世界空镜｜`y=1200`

`WK01 → WK02 → WK03 → WK04 → WK05`从左向右排列。

职责：建立世界层级；不得与场景关键帧、道具或角色卡混排。

### 02｜道具资产｜`y=2500`

`PROP01 → PROP02 → PROP03 → PROP04 → PROP05 → PROP06 → PROP07`从左向右排列。

职责：每件道具只保留一个画布媒体节点；新增道具继续向右追加，不另起重叠行。

### 03｜角色连续性

第一排`y=3900`：

1. `02-人物三状态连续性.png`
2. 小铭日常人物总卡
3. 小铭赛博小丑妆造卡
4. 日常24格表情卡
5. 小丑24格表情卡
6. F01—F04小丑表演补充卡

第二排`y=4800`：

`A65 → B65-F01 → B65-F02 → A85 → B85-F03 → C85-F04`

职责：第一排锁身份、妆造和表情；第二排锁65mm／85mm同机位切换。A/B/C板不得散入分镜或视频区。

### 04A｜S01—S04资料｜`y=6200`

从左向右：

1. `BOARD-A`四幕分镜板
2. KF01 v02当前场景工作副本
3. 新增纯文本节点：`[S01] 当前提示词＋Take02备注`
4. `INFO04` Seedance任务板
5. KF02酒馆主吧台
6. KF03日常小铭营业态
7. KF04记忆杯激活

新增文本节点固定包含：

```text
shot_id: PV01-S01
current_take: Take02
source: KF01 v02 / 零号酒馆
model: Seedance 2.0
preset: native1080p / 6s / 音频是 / 16:9
motion: 列车左→右、克制前移、轻微上仰、雨水和倒影持续运动
audio: 雨与列车环境层；无配乐、无对白
status: TECH-PASS / USER-REVIEW
continuation: 用户确认后才可成为S02父片
prompt: 当前S01完整生成提示词
```

### 06｜Seedance生成视频专属区｜`y=7600`

从左向右：

1. `INFO09`审片、连续性与交付边界
2. S01 Take02视频节点，显示名改为：`[REVIEW] S01-Take02｜零号酒馆入口｜1080p24｜6s｜有声`
3. 为S02预留空槽
4. 后续视频按镜号向右追加

KF01 v02 → S01视频的现有连线必须保留。视频区不放角色卡、道具卡、音频文件或导演预演。

### 04B｜S05—S08资料｜`y=9200`

`BOARD-B → KF06灯闪切换桥帧 → INFO05任务板`。

### 04C｜S09—S11资料｜`y=10300`

`BOARD-C → INFO06任务板`。

### 04D｜S12—S14资料｜`y=11400`

`BOARD-D → KF08片尾直视构图 → INFO07任务板`。

不同分镜资料不得混入同一行；后续实际场景帧、提示词或Take说明继续在对应分镜行向右追加。

### 05A｜对白音频｜`y=12800`

`VO01 → VO02 → … → VO13`按台词顺序向右排列。

### 05B｜环境／SFX／灯光｜`y=13700`

`INFO08 → AMB01 → SFX01 → SFX02 → SFX03 → SFX04`。

对白与环境音分行，避免查找口型音频时混入效果音。

### 07｜预演与交付｜`y=15100`

`导演分镜预演60s → 片尾海报帧`。

预演视频节点强制保留`ANIMATIC ONLY`含义，不得放入Seedance生成视频区。

### 99｜历史隔离｜`y=16400`

旧KF01 v01移动到历史区并改名为：

`[99历史] KF01-v01｜无零号酒馆正式招牌｜禁止当前调用`

历史区不得连接任何当前视频生成节点。

## 五、无重复上传与变更边界

### 允许

- 移动65个现有节点。
- 重命名现有节点显示名。
- 新建1个纯文本说明节点。
- 保留现有1条素材关联线。
- 调整画布视图、网格吸附和节点选择状态。

### 禁止

- 重新上传任何图片、音频、视频或信息板。
- 复制现有媒体节点作为视觉占位。
- 删除Take01本地历史文件或替换Take02视频内容。
- 触发新的Seedance生成或任何付费任务。
- 从历史区、预演区向当前生成视频建立连线。

### 数量闸门

- 整理前：65个节点、1条连线。
- 整理后：66个节点、1条连线。
- 唯一新增节点：`[S01] 当前提示词＋Take02备注`纯文本节点。
- 媒体节点总数不变；新增上传数必须为0。

## 六、移动与持久化方法

1. 先关闭所有节点参数展开状态并开启网格吸附。
2. 按区域自上而下分批移动，每批完成后读取节点逻辑坐标确认已持久化。
3. S01工作副本和视频节点最后移动，避免破坏现有连线。
4. 通过界面拖动完成位置变更，不以临时CSS修改伪装布局结果。
5. 所有节点移动完毕后重置视图，检查整体阅读顺序与空白区。

## 七、验收标准

### 数量与内容

- 66个节点、1条连线。
- 原65个节点ID全部仍存在。
- 新增媒体上传为0。
- S01 Take02输出URL、提示词和Seedance配置保持不变。

### 空间

- 节点静态边界相交数为0。
- 每个分类行从左向右递增，不出现回跳。
- 不同分类或分镜行的垂直距离满足坐标合同。
- 视频专属区、预演区和历史区互不混排。

### 语义

- PROP01—07全部在道具区。
- 5张角色主卡与6张A/B/C板全部在角色区。
- S01场景、当前提示词、任务板和Take状态可以在一个视野方向内找到。
- 每个已生成Seedance视频都有明确镜号、Take、参数、首帧、声音和审核状态。
- KF01 v01明确标为历史，KF01 v02保持当前首帧源。

## 八、风险处理

- 如果RunningHub拖动未持久化，立即停止下一批，刷新后验证该批坐标，不重复执行已成功移动的节点。
- 如果文字节点创建产生空白副本，保留有内容的唯一节点并删除空白副本；媒体节点不得删除。
- 如果S01关联线在移动后断开，先恢复原有KF01 v02 → S01视频关系，再继续其他整理。
- 如果节点展开面板遮挡邻区，只关闭面板，不改变已锁定的泳道坐标。
