## 二、架构与单一来源

### 数据流

```text

  → 项目人物画风规范
    → 人物资产卡
      → 角色造型差异配置

              → 24格表情卡
                → 人物模型关联包
```

### 职责边界

| 单元 | 唯一职责 | 不得承担 |
| --- | --- | --- |
| 世界观视觉圣经 | 时代、城市、空间、场景现实主义和全局文化边界 | 具体角色脸、服装或表情版本 |
| 项目人物画风规范 | 三人共享的渲染、国风结构、CG材质和电影光效合同 | 重写角色身份、年龄、身体、职业和道具 |
| 人物资产卡 | 角色正史、五官、身体、行为、服装、道具与连续性 | 复制整份项目画风正文 |
| 角色造型差异配置 | 角色专属轮廓、国风构件、琉璃几何、颜色和职业道具 | 改变共享风格比例 |
| 材质与色卡 | 可测量的厚度、粗糙度、透明度、折射、色值和光照响应 | 用空泛“高级感”替代物理描述 |
| 生成提示词 | 把正史、画风、变体、参考图职责和输出合同组装为一次生成任务 | 成为新的正史来源 |
| 人物关联包 | 当前有效文字、图片、版本和下游调用的唯一入口 | 自动加载候选图或历史图 |

共享画风与角色差异通过两个稳定接口分离：

```yaml
visual_style_id: "ZERO-ECHO-VIS-NEO-CHINESE-CYBER-GLAZE-02"
character_style_variant_id: "角色专属稳定编号"
```

`visual_style_id`回答“同一项目如何呈现”；`character_style_variant_id`回答“这个角色在共享画风里如何保持独特”。三名角色继续沿用原有`continuity_id`，不得因画风升级创建新角色身份。

## 三、共享画风合同

### 固定公式

`仿真人身份与人体结构50%＋新国风造型结构20%＋动漫化3D CG材质20%＋电影灯光与视效10%`

### 仿真人身份与人体结构｜50%

- 脸、天然眼睛、皮肤、头发、手、脚和身体比例保持真实摄影质感。
- 保留自然骨相、不对称、毛孔、绒毛、眉毛毛流、泪膜、唇纹、指节、甲床和真实关节受力。
- 年轻化只服从人物正史；不放大眼睛、不缩短中庭、不统一尖下巴、不磨成塑料皮肤。
- 服装风格不能覆盖身份参考、年龄、种族、伤痕、发际线、天然瞳色或核心道具。

### 新国风造型结构｜20%

- 国风来自领口与开合、榫卯受力、建筑比例、礼序、材质、留白和动作节奏。
- 传统结构必须被转译成可生产、可穿戴、可完成职业动作的近未来服装。
- 禁止龙凤贴图、巨型祥云、假古文堆砌、仙侠法器、宽袍拖地、影楼古装和国风COS。

### 动漫化3D CG材质｜20%

- 动漫感只作用于服装剪影、硬表面切面、线性高光、层次阴影和动作轮廓。
- 3D CG负责琉璃厚度与折射、织物纤维、金属磨损、皮革弯折、道具结构和真实投影。
- 禁止动漫脸、粗黑脸部描边、全机械四肢、重型机甲、悬浮玻璃碎片和全身透明化。

### 电影灯光与视效｜10%

- 人脸由4100—4300K柔和身份主光保护，天然肤色、眼白、虹膜与唇色不被青紫光污染。
- 青玉冷光和克制品红只描发丝、琉璃边缘、道具反射和外轮廓。
- 体积光、色差、镜头光晕和数据辉光只在材质折射或画面边缘出现。
- 场景仍服从情感现实主义，不能随人物资产板变成满屏霓虹夜店。

### 小铭造型合同

- 本人三张身份参考照继续是脸部最高优先级来源。
- 日常v04母样负责共享画风与日常零号酒保礼装；候选文件保持原像素和原校验值，不重新压缩。
- 小丑形态只改变固定全脸妆面与剧情灯光，不改变脸、发型、天然眼睛、基础礼装和琉璃连接结构。
- 人物卡重点展示调酒双手、斜襟榫卯、腰封挂点、短靴五视图和两件核心道具。

### 林野造型合同

- 视觉年龄保持25—27岁区间，黑色左分短发与一缕雨湿发固定。
- 脸为偏长方的椭圆骨相，颧骨轻、下颌清晰但不宽；不能被共享画风改成小铭同脸或成熟商务脸。
- 服装以“封闭、可预测、可计算”为视觉逻辑，所有数据片与42毫米存储环采用同心层规则。
- 人物总卡重点锁定同一林野身份、完整全身、正／侧／背三视图、右手存储环与服装安装位置；结构细节伴随卡重点证明竹简层、单层闭合、琉璃剖面和安装对应。
- 两张卡必须成对调用：总卡是脸、年龄、身体、发型、全身比例、三视图和道具方向的最高视觉正史；伴随卡是八层双片模块、单层闭合结构、材料剖面与安装对应的最高视觉正史。
- 当前总卡中错误的“竹简层”细节格、缺失的单层承重截面格和随机玻璃块“琉璃”格不得作为结构依据，由结构细节伴随卡明确覆盖；该覆盖不得反向改写总卡中的人物身份、全身或三视图。

### 姜禾造型合同

- 视觉年龄保持23—25岁区间，自然椭圆脸、柔和清楚的颧骨、干净但不尖的下颌固定。
- 黑棕及肩偏侧分发型固定，右侧别耳，左侧碎发允许随风运动。
- 服装以“开放、变化、保留呼吸”为视觉逻辑；相机肩带受力必须穿过服装结构而非悬浮。
- 人物卡重点展示相机负重路径、风衣动态、开放式领口、短披肩连接、动态发丝和珠光琉璃切片。

### 共享防同质化规则

1. 三人不得共享脸、眼距、鼻唇比例、下颌、发型、身高感或肩腰比例。
2. 三人不得共享领口、服装闭合、琉璃几何、识别色或核心道具。
3. 人物资产卡可共享信息架构，但每张卡的重点细节必须服从角色职业和人物弧。
4. 任何“同脸换发型”“同装换色”或“同一机甲模板换道具”的候选图均判定为阻断失败。

4. 文字正史锁定人物功能、行为、左右手和连续性。
5. 版式案例只提供区域布局，不继承其中人物、服装或道具。

### 林野

1. 林野v03总卡锁定同一脸、26岁年龄、身体、发型、服装连续性和存储环。
2. 人物资产卡与EP01正史锁定人物弧、动作、右手道具和青年感。
3. 小铭v04母样只提供项目共享画风，不得提供脸、身体、发型或服装身份。
4. 林野角色变体负责立领、竹简层、封闭几何和冷烟预测琉璃。
5. 林野v04总卡生成后，作为结构细节伴随卡的唯一视觉参考，只锁定人物身上已经成立的八层双片模块、石墨雨灰／预测青色系和冷烟预测琉璃材质；伴随卡不得复制完整人物脸或改写身份。

### 姜禾

1. 姜禾v03总卡锁定同一脸、24岁年龄、身体、发型、风衣连续性和相机。
2. 人物资产卡与EP01正史锁定人物弧、边界感、相机动作和青年感。
3. 小铭v04母样只提供项目共享画风，不得提供脸、身体、发型或服装身份。
4. 姜禾角色变体负责开放领口、飞檐弧线、短披肩和珠光流动琉璃。

### 人物总卡合同

- 人物总卡负责身份、完整全身、正侧背三视图、面部、发型、配饰、鞋履、核心道具、服装安装位置和色卡；不得被结构细节格挤压到无法稳定身份与三视图。
- 同一张卡内必须保持同脸、同年龄、同发型、同身体比例、同服装闭合、同道具尺寸和同左右手。
- 人物卡只使用简短准确中文标签；标签错误优先局部修正，不重新随机生成正确身份。
- 手部、足部、服装受力、配饰固定和道具握持属于阻断项，不能用版式美观掩盖。

### 林野结构细节伴随卡合同

- 固定3:4竖版且不出现完整人物脸；只以林野v04人物总卡锁定同一套八层双片模块、颜色和材质，不重新设计人物身份。
- 顶部45%为“竹简层”：八层、每层左右双片的精确正投影；中左为“安装对应”：用无脸简化衣装轮廓标出立领下缘至前襟分叉，以及侧面、背面的对应安装；中右为“单层闭合”：同一层左右双片、中央黑钛承重轨、铰接和闭合截面；底部为“琉璃剖面”和“色卡”。
- “琉璃剖面”必须取同一片窄矩形模块，显示2.5—6毫米烟灰主体、预测青内层、黑钛包边和极窄品红倒角；禁止随机玻璃块。
- 只允许准确文字“林野”“结构细节伴随卡”“竹简层”“安装对应”“单层闭合”“琉璃剖面”“色卡”；不得出现数字、编号、伪字、微小段落或其他标签。
- 伴随卡是竹简层、安装对应、单层闭合和琉璃剖面的最高视觉正史，并覆盖总卡中错误的“竹简层”格、空白单层结构格和随机玻璃块“琉璃”格；总卡仍是身份、全身与三视图的最高视觉正史。

### 表情卡合同

- 对应人物总卡或妆造卡通过审核后才能生成表情卡。
- 固定4列×6行24格；每格同时显示两位编号与准确中文情绪名。
- 24格使用同一脸、年龄、发际线、发型、天然眼睛、基础领口、固定配饰、背景和身份光。
- 情绪由眉峰、眉间、上下眼睑、视线、鼻翼、嘴角、唇线、下颌、呼吸和泪膜共同完成，禁止只换嘴形。
- 小铭小丑卡的妆面几何在24格中必须固定；泪水和表情不得随机冲掉妆纹。

## 八、分阶段迁移流程

### 阶段A｜建立文字框架

1. 创建v04项目画风规范、角色差异矩阵和三张角色材质卡。
2. 给6份通用模板增加`character_style_variant_id`接口。
3. 升级15份角色文字资产，写入候选v04路径、共享画风编号和角色变体编号。
4. 此阶段关联包继续指向v03；文字状态使用`v04迁移中`或`待生成v04`，不得写成已锁定。

### 阶段B｜完成四张基础身份卡与林野结构伴随卡

1. 复核小铭现有v04母样身份与校验值，准备无损正式副本。
2. 生成小铭v04小丑妆造卡，审核同脸、妆纹、礼装和天然眼睛。
3. 生成林野v04总卡，审核26岁青年感、存储环、封闭结构和非霸总化。
4. 以通过身份／三视图审核的林野v04总卡生成独立结构细节伴随卡，审核八层双片、安装对应、单层闭合和同片琉璃剖面。
5. 生成姜禾v04总卡，审核24岁青年感、相机受力、开放结构和非时装模特化。
6. 任何身份、年龄、手脚、闭合或道具错误均先保存失败样张，再用同一身份锚点定向修正；林野总卡中的三个错误结构细节格由伴随卡覆盖，不再以破坏身份／三视图为代价继续重生总卡。

### 阶段C｜生成四张表情卡

1. 只使用阶段B通过审核的对应总卡或妆造卡作为造型源。
2. 生成小铭日常、小铭小丑、林野、姜禾四张24格表情卡。
3. 逐卡校验01—24编号、中文名、同脸、年龄、发型、领口、妆纹、天然眼眸和剧情情绪区分。

### 阶段D｜原子切换

1. 在移动v03前确认9张v04均存在、非空、可解码并通过技术审核。
2. 记录三个关联包的v03当前路径、文件尺寸和SHA-256，形成迁移清单。
3. 一次性把三个人物关联包、人物卡和索引的当前入口切到v04。
4. 把8张v03移入各角色`历史版本/`并写入历史表；不得删除。
5. 原子切换后仍统一保持`待审`，只有用户最终确认视觉结果才改为`锁定`。

## 九、失败处理与回滚

| 失败类型 | 处理 | 是否允许进入当前关联包 |
| --- | --- | --- |
| 身份漂移、年龄错误或同脸化 | 保留失败样张并标注原因；沿原参考优先级定向重生或编辑 | 否 |
| 手脚畸形、道具方向错误、服装不闭合 | 使用当前身份图做局部修正；不可用裁切隐藏错误 | 否 |
| 中文标签错字但人物正确 | 只编辑对应文字区域，其他画面完全不变 | 修正后复核 |
| 琉璃塑料化、全身机甲化或青紫染脸 | 按角色材质卡局部重绘，恢复真实厚度、支撑和身份光 | 否 |
| 表情卡变脸、重复表情或妆纹漂移 | 保留正确格，局部重绘错误格；不得以错误表情卡扩散分镜 | 否 |
| 原子切换后校验失败 | 依据迁移清单恢复三个关联包的v03路径；v04降回候选待修 | 否 |

失败图不删除，但必须进入历史或候选记录并明确`废弃`、`禁止下游调用`。没有通过同脸、年龄、手脚、道具、标签和连续性检查的图片，不得仅因“整体好看”而成为当前资产。
