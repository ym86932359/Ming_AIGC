# 零点回声酒馆 - v04角色卡候选设计

## 目标与范围

根据当前Obsidian最新人物正史重新生成8张v04候选资产：

1. 小铭日常赛博酒保人物总卡。
2. 小铭赛博小丑妆造卡。
3. 林野人物总卡。
4. 姜禾人物总卡。
5. 小铭日常赛博酒保24格面部表情卡。
6. 小铭赛博小丑24格面部表情卡。
7. 林野24格面部表情卡。
8. 姜禾24格面部表情卡。

本轮同步生成角色卡与表情库，但不修改当前v03图片、不切换人物关联包。v04只作为全新的国漫CG视觉候选，用户确认后再另行决定是否替换当前正式资产。

## 统一画风合同

- 风格编号：`ZERO-ECHO-VIS-CYBER-GLAZE-CG-02`。
- 画面比例：真人身份与成年骨相40%＋国漫CG脸部与皮肤25%＋动画电影结构光影15%＋赛博琉璃材质12%＋电影炫光8%。
- 人脸采用高端国漫3D动画电影CG：保留可辨识身份、成年骨相和自然面部不对称，同时强化干净的面部切面、柔和的鼻唇沟过渡与清晰的眉眼轮廓。
- 皮肤采用物理渲染的国漫CG材质：受控次表面散射、细腻微纹理、柔和绒毛和动画电影式高光滚降；不能变成塑料娃娃、陶瓷面具或完全摄影写实皮肤。
- 发型采用成束梳理的CG发丝结构并保留少量独立飞发；天然深棕虹膜、黑色瞳孔、自然眼白和湿润角膜高光必须始终存在。
- 青紫炫光只描出发丝、琉璃边缘和人物外轮廓，不覆盖肤色、眼白、虹膜和唇色。
- 角色卡使用3:4竖版，包含主形象、三视图、面部六格、发型、服装、鞋履、配饰、道具、色卡和呈现参数。
- 图内只保留准确的简体中文标题与短标签，不出现导演、镜头、场景、选角或长段说明。

## 表情库合同

- 四张表情卡全部使用3:4竖版、4列×6行、共24格的固定网格；每格保留编号、对应情绪名称和统一头肩裁切。
- 表情差异必须由眉峰、上下眼睑压力、视线、鼻翼、嘴角、下颌、呼吸感和泪膜共同表达，不能用换脸、换发型、换服装或夸张颜艺代替。
- 同一角色的24格必须继承对应v04人物总卡的同一张脸、同一国漫CG皮肤着色、同一视觉年龄、发际线、天然虹膜、领口、配饰与连续性标记。
- 小铭日常卡24格全部保持日常酒保妆造；小铭小丑卡24格全部保持同一套完整小丑妆，不在情绪之间改变妆纹几何、颜色或覆盖范围。
- 情绪标签服务后续分镜调用，必须与人物表情提示词文件中的01—24编号逐项一致；不得合并、漏项、重复或使用含义模糊的标签。
- 国漫CG不是卡通夸张：禁止巨眼、尖下巴、二次元平涂、手游立绘脸、西方卡通比例、塑料磨皮和网游角色渲染。

## 角色差异

### 小铭日常

- 身份最高来源为三张本人照片，当前v03日常总卡只补身体、发型、服装和道具连续性。
- 使用黑曜青玉琉璃、旧金结构骨架和暗朱砂识别色。
- 保持中式非对称酒保轮廓、黑钛调酒匙、记忆杯和酒红擦杯巾。

### 小铭小丑

- 以三张本人照片和v04日常卡锁定同脸，以v03小丑妆造卡锁定妆面几何。
- 全脸暖象牙底、墨黑眼翼、朱砂鼻与笑弧、旧金额纹和烟青玉电路保持固定。
- 妆是绘制在真实皮肤上的妆层，不变成面具、玻璃脸、伤口或现成小丑IP。

### 林野

- 26岁、视觉年龄25—27岁；保持左分短发、青年成年骨相和右手未来存储环。
- 使用冷烟预测晶体、窄长矩形、平行切面与同心数据层。
- 禁止成熟霸总化、三十岁以上视觉年龄、金色东方纹样和全身发光西装。

### 姜禾

- 24岁、视觉年龄23—25岁；保持黑棕及肩偏侧分发、右侧别耳、旧银耳钉和纪实相机。
- 使用珠光流动琉璃、开放弧面、柔和折线和轻透明层。
- 禁止成熟时装模特化、婚纱感、红唇、高跟鞋和整件风衣玻璃化。

## 参考优先级

1. 用户本人照片或当前角色v03身份卡。
2. 当前人物资产卡中的年龄、骨相、服装、道具和连续性正史。
3. 当前v03图片的正侧背结构。
4. 已锁定赛博琉璃视觉规范与风格预览。
5. 通用角色卡版式模板。

低优先级来源不得覆盖高优先级身份、年龄、天然眼眸、妆面和道具方向。

## 验收门槛

- 八张图片均可解析并采用3:4竖版；人物总卡的主形象与三视图不裁脚。
- 四张表情卡必须严格为4×6共24格，情绪编号、中文名称和顺序与对应提示词逐项一致。
- 同一张卡内主形象与正侧背同脸、同年龄、同发型、同服装。
- 同一人物的总卡与表情卡同脸、同视觉年龄、同发型、同国漫CG皮肤材质和同服装版本。
- 小铭日常与小丑同脸；小丑妆纹在正面、三分之二和侧面闭合。
- 林野保持25—27岁视觉年龄，姜禾保持23—25岁视觉年龄。
- 琉璃具有厚度、支撑、折射、投影和真实连接点，不悬浮、不铺满身体。
- 手部、鞋履、相机、存储环、调酒匙和记忆杯结构可用。
- 中文标题与必要短标签清楚；不出现乱码、英文段落、品牌或水印。
- 所有v04候选保持`待审`，不自动替换或覆盖当前v03。
