## 维护规则

1. 剧本、镜头或任务变更时，先更新正史Markdown，再运行`build-runninghub-boards.mjs`。
2. 新板写入新版本目录，不覆盖已经用于生产或上传的旧板。
3. 只有`ACCEPTED`成片或实际末帧可以成为下一任务参考；`RETAKE`、`REJECTED`和`ANIMATIC ONLY`不得接生成下游。
4. 真人参考照、候选版本、历史版本、KF05、KF07和失败Take继续保持隔离。
